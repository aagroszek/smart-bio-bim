Create a free Supabase project

Go to supabase.com → New Project → free tier. Create a table called bim_instances with columns: model text, zone text, qty int, phase text, source text. Copy your Project URL and anon key from Settings → API.

2️⃣
Paste your Supabase credentials here

Enter your URL and key below — the dashboard will then poll Supabase every 5 seconds and auto-update whenever Rhino writes a new count.

3️⃣
Run the Grasshopper script in Rhino

Download the rhino_tracker.gh file (button below). Open it in Grasshopper. It detects all objects on the layer MyceliumPanel, counts them, and POSTs the count to Supabase every time the count changes. The dashboard updates automatically within 5 seconds.

4️⃣
Or use the Revit Dynamo script

For Revit: run the mycelium_tracker.dyn Dynamo script (download below). It queries all instances of the MyceliumPanel family and POSTs the count to Supabase on run, or scheduled via the Dynamo Player.

How live tracking works

Data flow between Rhino / Revit and this dashboard
🔴
Why Rhino doesn't auto-update this dashboard

The dashboard is a static HTML file. When you copy a panel in Rhino, Rhino has no way to "call" a webpage on its own — there's no built-in connection. A bridge is needed: a script running inside Rhino that detects changes and sends data out, and a small database that the dashboard reads from.

🟡
Manual sync (works right now)

Click "+ Sync from Rhino / Revit" in the top bar and type the filename + instance count. This instantly updates the dashboard totals, impact data, and Myra's awareness. Good for presentations or small teams.

🟢
Automatic live sync (full setup ~20 min, free)

Stack: Rhino Grasshopper Python script → Supabase free database → this dashboard polls every 5 seconds.

Once set up: every time you copy/delete a panel object on the MyceliumPanel layer in Rhino, the Grasshopper script fires, updates Supabase, and the dashboard refreshes automatically — showing live instance count, carbon total, cost, and install hours.
